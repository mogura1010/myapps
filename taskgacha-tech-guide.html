1HTML — 骨格（何を置くか）

一言で言うと：画面に「何を置くか」をタグで宣言するもの。

基本の書き方

① CSS変数（テーマ管理）

3JavaScript — ロジック（どう動くか）

一言で言うと：「ボタンを押したら何かする」という動きを書くもの。ガチャの確率計算・画面更新・データ管理もすべてここ。

① ボタンに動きをつける

// ボタンを押したときの処理を登録

② ガチャの確率ロジック

0〜1の乱数を生成して、確率帯に当てはめています。

// レア度と確率の定義
const RATES = [
  ["N",   0.60],  // 60%
  ["R",   0.27],  // 27%
  ["SR",  0.10],  // 10%
  ["SSR", 0.03],  //  3%
];

function drawRarity() {
  const r = Math.random();  // 0.0〜1.0 のランダムな数
  let c = 0;
  for (const [rarity, prob] of RATES) {
    c += prob;
    if (r < c) return rarity;  // 確率帯に入ったら返す
  }
}

例：乱数が 0.72 なら → N(0.60)を超えている → R(0.87)以下 → R確定

4localStorage — ブラウザ内保存

一言で言うと：ブラウザに付属している「小さなメモ帳」。ページを閉じても消えない。

基本の読み書き

// 保存（オブジェクト → JSON文字列に変換して保存）
localStorage.setItem("taskGachaData_v2", JSON.stringify(data));

// 読み込み（JSON文字列 → オブジェクトに戻す）
const data = JSON.parse(localStorage.getItem("taskGachaData_v2"));

JSON とは？

データを文字列として保存・送受信するための書き方です。

// JavaScriptのオブジェクト（プログラム内の形）
const data = {
  points: 150,
  tasks: [{ name: "新規ケース対応", pt: 20 }]
};

// JSON文字列（保存・バックアップファイルの中身）
'{"points":150,"tasks":[{"name":"新規ケース対応","pt":20}]}'

なぜバックアップが必要？

localStorage はファイルのパスごとに別管理 C:\OneDrive\MyApps\taskgacha.html → データA（本番） C:\Documents\ai-test\taskgacha.html → データB（別扱い） ブラウザのキャッシュクリアでも消える可能性あり → 定期的に「📤 バックアップを書き出す」でJSONを保存
💡 ポイント
localStorageはシンプルで手軽ですが「その端末・そのブラウザ専用」です。複数端末で同期したい場合はサーバー（データベース）が必要になります。

5AI画像生成 — カード画像の作り方

これだけ毛色が違います。Gemini AIにプロンプトを送り、返ってきた画像をHTMLに直接埋め込む流れです。

全体の流れ

① プロンプトをJSONファイルに書く 「縦型カード、渋谷を背景に勇者が剣を振る、アニメ調...」 ↓ ② nano-banana スキルがGemini APIを呼び出す ↓ ③ WebP画像ファイルが生成される（高画質・大きめ） ↓ ④ Python（pillow）で圧縮（512px・品質80） 9.5MB → 0.1MB 程度に削減 ↓ ⑤ Base64に変換（画像→文字列） ↓ ⑥ HTMLに直接埋め込み（外部ファイル不要！）

Base64とは？

画像データを「文字列」に変換する方法です。これのおかげでHTMLファイル1個の中に18枚の画像が全部入っています。

<!-- 通常の画像（外部ファイルが必要） -->
<img src="card01.webp">

<!-- Base64埋め込み（ファイル1個で完結） -->
<img src="data:image/webp;base64,UklGRi...">
<!--                              ↑ ここに長い文字列が続く -->

確率の仕組みを図で理解する

Math.random() は 0.0〜1.0 のランダムな小数を返します。これを「確率の帯」に当てはめることでレア度を決めます。

// 0.0 ───────────────────────────── 1.0
//  |←── N(60%) ──→|← R(27%)→|←SR→|SSR|
//  0.0            0.60      0.87  0.97 1.0

実際のコード（アプリから抜粋）

const RATES = [
  ["N",   0.60],
  ["R",   0.27],
  ["SR",  0.10],
  ["SSR", 0.03],
];

function drawRarity() {
  const r = Math.random();  // 例: 0.723
  let c = 0;
  for (const [rarity, prob] of RATES) {
    c += prob;               // c が 0.60 → 0.87 → 0.97 → 1.0 と増える
    if (r < c) return rarity; // 初めて r を超えた時点でそのレア度を返す
  }
}

1行ずつ読み解く

  • 1Math.random() で 0.723 などの数が出る
  • 2ループで c に確率を足していく（0.60 → 0.87 → 0.97 → 1.0）
  • 30.723 は 0.60 より大きい → N は外れ
  • 40.723 は 0.87 より小さい → R が当選

カードプールから1枚選ぶ

// レア度が決まったら、そのプールからランダムに1枚
function pick(arr) {
  return arr[Math.floor(Math.random() * arr.length)];
}

const rarity = drawRarity();               // 例: "R"
const pool = CARD_POOL.filter(c => c.rarity === rarity);
const card = pick(pool);                    // Rカードの中から1枚
💡 ポイント
Math.floor() は小数点以下を切り捨てる関数。Math.random() * 5 で 0.0〜4.99…が出るので、floorすると 0〜4 の整数になります。配列の添え字（番号）として使えます。

① アロー関数 vs 通常関数

アプリ内では () => {} という書き方を多用しています。

// 通常の関数（古い書き方）
function add(a, b) { return a + b; }

// アロー関数（モダンな書き方・ほぼ同じ意味）

// イベントで使う例（アプリのボタン処理）

② 配列メソッド（filter / map / find）

アプリ内でカードを絞り込むときに使っています。

const cards = [
  { id: "c01", rarity: "N"  },
  { id: "c02", rarity: "SR" },
  { id: "c03", rarity: "N"  },
];

// filter：条件に合うものだけ残す
const nCards = cards.filter(c => c.rarity === "N");
// → [{id:"c01",...}, {id:"c03",...}]

// map：各要素を変換して新しい配列を作る
const ids = cards.map(c => c.id);
// → ["c01", "c02", "c03"]

// find：最初に条件に合うものを1つ返す
const sr = cards.find(c => c.rarity === "SR");
// → {id:"c02", rarity:"SR"}

③ 非同期処理（async / await）

AI画像生成やAPIとの通信は「時間がかかる処理」です。JSはデフォルトで待たずに次へ進むため、await で「完了するまで待つ」と明示します。

// ❌ 待たないと結果が空になる
const data = fetch("https://api.example.com/cards");
console.log(data); // → Promise（まだ取得中）

// ✅ await で完了を待つ
async function loadCards() {
  const res  = await fetch("https://api.example.com/cards");
  const data = await res.json();
  console.log(data); // → [{...}, {...}] ちゃんと取れる
}

localStorageとは何か

ブラウザが持つ「キーと値のペアを保存するメモ帳」です。ページを閉じても消えません。容量は約5MB。

// 保存（値は必ず文字列）
localStorage.setItem("myKey", "hello");

// 読み込み
localStorage.getItem("myKey");  // → "hello"

// 削除
localStorage.removeItem("myKey");

JSONとは何か

オブジェクト（複雑なデータ）を「文字列」に変換するルール。localStorageは文字列しか保存できないので必須です。

// オブジェクト（JS内のデータ）
const data = { points: 150, log: [{ name: "ケース対応", pt: 20 }] };

// JSON文字列に変換（保存用）
const str = JSON.stringify(data);
// → '{"points":150,"log":[{"name":"ケース対応","pt":20}]}'

// JSON文字列をオブジェクトに戻す（読み込み用）
const obj = JSON.parse(str);
// → { points: 150, log: [...] }

アプリの保存・読み込みの流れ

  • 1タスク完了 → data.points += 20 でオブジェクトを更新
  • 2JSON.stringify(data) で文字列化
  • 3localStorage.setItem("taskGachaData_v2", ...) で保存
  • 4次回起動時に getItem → JSON.parse で復元

バックアップJSONの仕組み

// 書き出し：localStorageの中身をファイルとしてダウンロード
function exportBackup() {
  const json = JSON.stringify(data, null, 2); // 2=見やすくインデント
  const blob = new Blob([json], { type: "application/json" });
  const url  = URL.createObjectURL(blob);
  const a   = document.createElement("a");
  a.href = url;  a.download = "taskgacha-backup.json";
  a.click();  // ダウンロード開始
}

① ダークモード自動切り替え

OSの設定を検知して色を自動で変えます。CSS変数と組み合わせることで、1箇所書き換えるだけで全体に反映されます。

/* ライトモード（デフォルト） */
:root {
  --bg: #f7f6f2;
  --text: #222;
  --accent: #6c4ad6;
}

/* ダークモード（OSがダークのとき自動適用） */
@media (prefers-color-scheme: dark) {
  :root {
    --bg: #1c1c1a;    /* 同じ変数名で上書きするだけ */
    --text: #f2f1ec;
    --accent: #9b7df0; /* 暗い背景に合わせ少し明るく */
  }
}

/* あとは var() で使うだけ。ライト/ダーク自動対応 */
.card { background: var(--bg); color: var(--text); }

② カードフリップアニメーション

@keyframes でアニメーションの動きを定義し、要素に animation で適用します。

/* ① アニメーションの動きを定義 */

  0%   {

    transform: rotateY(0deg)  scale(1);
    opacity: 1;  /* 正面 & 等倍 & 不透明へ */

/* ② 要素に適用（.6s = 0.6秒かけて） */

}

③ SSRのホログラム演出

グラデーションをアニメーションで動かすことでキラキラ感を出しています。

@keyframes holo {
  0%   { background-position: 0%   50%; }
  50%  { background-position: 100% 50%; }
  100% { background-position: 0%   50%; }
}

④ Gridレイアウトの仕組み

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(130px, 1fr));
  /*                       ↑自動で列数を決める  ↑最小130px、余りは均等 */
  gap: 11px; /* カード間の隙間 */
}

画面が広ければ4列、狭ければ2列と自動調整されます。レスポンシブデザインの代表的な書き方です。

💡 cubic-bezier とは？
アニメーションの「速度の変化」を制御する関数。cubic-bezier(.2,.8,.3,1) は最初ゆっくり・中盤速く・最後に少しバウンドする動きです。cubic-bezier.com でビジュアルで試せます。

疑似要素（::before / ::after）

HTMLを追加せずにCSSだけで装飾を追加できます。

/* ボタンの左に装飾を追加する例 */
.btn::before {
  content: "✨";      /* 表示する内容 */
  margin-right: 6px; /* 本文との隙間 */
}

CSS変数の計算への応用

/* 変数同士を calc() で計算できる */
:root { --spacing: 8px; }

.card {
  padding: calc(var(--spacing) * 2);  /* = 16px */
  margin-bottom: var(--spacing);         /* = 8px */
}
/* --spacing を変えるだけで全体の余白が一括変更される */

transition（スムーズな状態変化）

.btn {
  background: var(--accent);
  transition: background .15s, transform .05s;
  /* ↑ 変化するプロパティと所要時間を指定 */
}
.btn:hover  { background: var(--accent-d); }  /* ホバーで暗く */
.btn:active { transform: scale(.96); }       /* 押すと縮む */

よくあるミス①：詳細度の競合

⚠️ スタイルを書いたのに反映されない
/* 詳細度（Specificity）: idが最も強い */
#header .btn { color: red; }    /* 詳細度: 高 */
.btn          { color: blue; }  /* 詳細度: 低 → 負ける */

/* 確認方法: ブラウザのDevToolsで打ち消し線が入っているCSSを探す */
/* 対策: idセレクタは使わず class で統一する */

よくあるミス②：aspect-ratio が効かない

⚠️ カードが縦長にならず潰れる
/* aspect-ratio は幅が決まっていないと機能しないことがある */
.mini {
  aspect-ratio: 3/4;   /* ← これだけでは不十分な場合がある */
  width: 100%;         /* ← 幅を明示するとちゃんと効く */
}

  • 2右パネルの「Styles」で適用されているCSSを確認
  • 3打ち消し線が入ったCSSが「負けているスタイル」
  • 4「Computed」タブで実際に計算された最終値を確認

Base64の仕組み

画像（バイナリデータ）を「テキスト文字だけ」で表現する変換方式です。A〜Z / a〜z / 0〜9 / + / / の64文字だけを使います。

// 通常：画像は外部ファイルを参照
<img src="card01.webp">  ← ファイルがないと表示されない

// Base64：データをそのまま文字列として埋め込む
<img src="data:image/webp;base64,UklGRi...">
//                              ↑ 数万文字の文字列が続く

圧縮が必要な理由

  • 1AIが生成する画像は高解像度で 1枚あたり約0.5〜1MB
  • 218枚 × 1MB = 約18MB → ブラウザが重くなる
  • 3Pythonのpillowで512px・WebP品質80に圧縮 → 1枚 約40KB
  • 418枚 × 40KB = 約0.7MB → 問題なし ✅

Pythonでの圧縮コード（参考）

from PIL import Image

img = Image.open("card01_original.webp")

# 縦横比を保ちつつ512px以内にリサイズ
img.thumbnail((512, 512))

# WebP形式で品質80で保存（ファイルサイズ激減）
img.save("card01.webp", "WEBP", quality=80)
💡 WebPとは？
Googleが開発した画像フォーマット。JPEGと同じ画質でファイルサイズが約25〜34%小さくなります。モダンなブラウザはすべて対応済みで、Webアプリに最適です。

  3:4 portrait aspect ratio,

  place name label clearly displayed on card.";

// 個別カード固有の部分だけ変える
const prompt = BASE + ` CHARACTER: ${job}. POSE: ${pose}.
  SETTING: ${place}. ART STYLE: ${style}.`;

よくあるミス①：Base64が壊れて画像が表示されない

⚠️ 画像が表示されず壊れたアイコンになる
// 原因: 改行・スペースが混入している
// ❌ Base64文字列に改行が入るとブラウザが認識できない
src="data:image/webp;base64,UklGR
i5CAAAB..."  // ← 途中に改行 NG

// ✅ Pythonで正しく変換する方法
import base64
with open("card.webp", "rb") as f:
    b64 = base64.b64encode(f.read()).decode("utf-8")
    # decode("utf-8") → 改行なしの純粋な文字列になる

よくあるミス②：Windowsで日本語のUnicodeエラー

⚠️ UnicodeDecodeError: 'cp1252' codec can't decode byte
# 原因: WindowsのPythonはデフォルトでcp1252（欧文）で読む
# ✅ 対策①: JSONに日本語を入れず ensure_ascii=True で保存
import json
data = { "place": "渋谷" }
json_str = json.dumps(data, ensure_ascii=True)
# → {"place": "渋谷"}  ← ASCII文字だけになる

# ✅ 対策②: ファイルを開くとき encoding 指定
with open("config.json", encoding="utf-8") as f:
    data = json.load(f)
